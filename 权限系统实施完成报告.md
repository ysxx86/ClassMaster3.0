## 技术实现细节

### 1. 前后端双重验证

**前端验证** (用户体验优化)
- 根据权限动态显示/隐藏菜单
- 禁用无权限的操作按钮
- 提供友好的权限提示

**后端验证** (安全保障)
- 装饰器验证API访问权限
- 数据查询时自动过滤
- 返回403错误给越权请求

### 2. 基于角色的访问控制 (RBAC)

**角色定义**
- 超级管理员 (is_admin = 1)
- 正班主任 (primary_role = '正班主任')
- 副班主任 (primary_role = '副班主任')
- 科任老师 (primary_role = '科任老师')
- 行政 (primary_role = '行政')
- 校级领导 (primary_role = '校级领导')

**权限检查函数**
```python
# 角色判断
is_super_admin(user)
is_head_teacher(user)
is_vice_teacher(user)
is_subject_teacher(user)
is_admin_staff(user)
is_school_leader(user)

# 功能权限
can_access_students(user, class_id)
can_edit_grade(user, class_id, subject)
can_access_deyu(user, class_id)
can_access_comments(user, class_id)
can_access_performance(user)

# 数据过滤
get_accessible_classes(user)
get_editable_subjects(user, class_id)
```

**权限装饰器**
```python
@require_admin  # 要求超级管理员
@require_head_teacher  # 要求正班主任或超级管理员
@require_performance_access  # 要求班主任考核权限
```

### 3. 数据过滤机制

**班级过滤**
- 超级管理员: 查询所有班级
- 正班主任: 只查询自己的班级
- 其他角色: 返回空列表

**学科过滤**
- 超级管理员: 所有学科
- 正班主任: 自己班级的所有学科
- 其他教师: 只有自己任教的学科

**SQL查询示例**
```python
# 学生管理 - 根据权限过滤班级
accessible_classes = get_accessible_classes(current_user)
placeholders = ','.join('?' * len(accessible_classes))
cursor.execute(f'''
    SELECT s.*, c.class_name 
    FROM students s
    LEFT JOIN classes c ON s.class_id = c.id
    WHERE s.class_id IN ({placeholders})
    ORDER BY CAST(s.id AS INTEGER)
''', accessible_classes)
```

## 安全特性

### 1. 防止越权访问
- API层面验证用户权限
- 数据库查询自动添加过滤条件
- 前端隐藏无权限的功能入口

### 2. 会话管理
- 用户登录时获取权限信息
- 权限信息存储在session中
- 角色变更需要重新登录生效

### 3. 日志记录
- 记录权限检查结果
- 记录越权尝试
- 便于安全审计

### 4. 错误处理
- 统一的403错误响应
- 友好的错误提示信息
- 不泄露敏感信息
