# 科任老师多班级显示修复说明

## 问题描述

科任老师登录后，成绩管理页面只显示一个班级的学生。但科任老师可能任教多个班级，需要显示所有班级的学生以便进行成绩评定。

## 问题原因

在 `grades.py` 的 `get_all_grades()` API 中，非管理员用户被限制为只能查看自己的 `class_id`：

```python
# 原来的代码
if not current_user.is_admin:
    if not current_user.class_id:
        return jsonify({...})
    class_id = current_user.class_id  # 所有非管理员都被限制为自己的班级
```

这个逻辑对正班主任是正确的（只管理自己班级），但对科任老师、副班主任、行政、校级领导是错误的（需要看到所有班级）。

## 修复方案

### 1. 后端修改 (`grades.py`)

修改了所有成绩相关的 API 端点，使用 `is_head_teacher()` 函数来区分不同角色：

#### 修改的 API 端点：
- `get_all_grades()` - 获取成绩列表
- `save_student_grade()` - 保存学生成绩
- `delete_student_grade()` - 删除学生成绩
- `preview_grades_import()` - 预览成绩导入
- `confirm_grades_import()` - 确认成绩导入
- `import_grades()` - 导入成绩
- `download_grades_template()` - 下载成绩模板

#### 权限逻辑：

```python
from utils.permission_checker import is_head_teacher

if not current_user.is_admin:
    if is_head_teacher(current_user):
        # 正班主任：只能查看自己的班级
        if not current_user.class_id:
            return jsonify({'status': 'error', 'message': '您尚未被分配班级'})
        class_id = current_user.class_id
    # 其他角色（科任老师、副班主任、行政、校级领导）：
    # class_id 保持为空字符串，表示查询所有班级
```

### 2. 前端修改 (`js/grades.js`)

修改了前端 JavaScript，在保存成绩时传递 `class_id` 参数：

#### 修改的函数：
- `renderGradesTable()` - 渲染表格时添加 `data-class-id` 属性
- `updateGrade()` - 保存单个成绩时获取并传递 `class_id`
- `setAllGradesExcellent()` - 批量设置"优"时传递 `class_id`
- `clearAllGrades()` - 批量清空成绩时传递 `class_id`
- `processPastedText()` - 粘贴成绩时传递 `class_id`

#### 关键修改：

```javascript
// 在渲染表格时添加班级ID属性
row.setAttribute('data-class-id', studentGrade.class_id);

// 在保存成绩时获取班级ID
const row = selectElement.closest('tr');
const classId = row ? row.getAttribute('data-class-id') : null;

// 在请求数据中包含班级ID
const gradeData = {
    semester: currentSemester,
    class_id: classId,
    [subject]: value
};
```

## 修复后的权限规则

| 角色 | 成绩管理 - 查看范围 | 成绩管理 - 编辑权限 |
|------|-------------------|-------------------|
| 超级管理员 | ✅ 所有班级 | ✅ 所有学科 |
| 正班主任 | ✅ 自己班级 | ✅ 本班级所有学科 |
| 副班主任 | ✅ 所有班级 | ✅ 自己任教的学科 |
| 科任老师 | ✅ 所有班级 | ✅ 自己任教的学科 |
| 行政 | ✅ 所有班级 | ✅ 自己任教的学科 |
| 校级领导 | ✅ 所有班级 | ✅ 自己任教的学科 |

## 注意事项

1. 前端已经实现了学科权限控制，非本学科的成绩输入框会被禁用（灰色显示，无法点击）
2. 后端也有双重验证，即使前端被绕过，后端也会检查权限
3. 所有成绩保存操作现在都需要传递 `class_id` 参数
4. 正班主任的 `class_id` 会自动从用户信息中获取，其他角色需要从前端传递
