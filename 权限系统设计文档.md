## 详细权限说明

### 1. 超级管理员
**权限范围**：系统所有功能
- 用户管理
- 班级管理
- 学科管理
- 教师分配
- 所有班级的学生管理
- 所有班级的成绩管理
- 所有班级的评语管理
- 成绩分析
- 班主任考核
- 系统设置

### 2. 正班主任
**权限范围**：自己班级的完整管理权限
- ✅ 查看和管理自己班级的学生
- ✅ 录入和修改自己班级所有学科的成绩
- ✅ 管理自己班级的德育数据
- ✅ 编写和导出自己班级的评语
- ✅ 查看成绩分析
- ✅ 导出自己班级的报告
- ❌ 不能访问其他班级数据
- ❌ 不能访问班主任考核
- ❌ 不能访问管理后台

### 3. 副班主任
**权限范围**：仅限自己任教学科的成绩录入
- ✅ 录入和修改自己任教学科的成绩（所有班级）
- ✅ 查看成绩分析
- ❌ 不能管理学生
- ❌ 不能录入其他学科成绩
- ❌ 不能管理德育数据
- ❌ 不能编写评语
- ❌ 不能访问班主任考核

### 4. 科任老师
**权限范围**：仅限自己任教学科的成绩录入
- ✅ 录入和修改自己任教学科的成绩（所有班级）
- ✅ 查看成绩分析
- ❌ 不能管理学生
- ❌ 不能录入其他学科成绩
- ❌ 不能管理德育数据
- ❌ 不能编写评语
- ❌ 不能访问班主任考核

### 5. 行政
**权限范围**：成绩录入 + 班主任考核
- ✅ 录入和修改自己任教学科的成绩（如果有任教）
- ✅ 查看成绩分析
- ✅ 访问班主任考核功能
- ❌ 不能管理学生
- ❌ 不能管理德育数据
- ❌ 不能编写评语

### 6. 校级领导
**权限范围**：成绩录入 + 班主任考核
- ✅ 录入和修改自己任教学科的成绩（如果有任教）
- ✅ 查看成绩分析
- ✅ 访问班主任考核功能
- ❌ 不能管理学生
- ❌ 不能管理德育数据
- ❌ 不能编写评语

## 权限判断逻辑

### 菜单显示逻辑
```javascript
function getVisibleMenus(userRole, userClassId, userSubjects) {
    const menus = {
        home: true,  // 所有人都能看到首页
        students: false,
        grades: false,
        deyu: false,
        comments: false,
        gradeAnalysis: true,  // 所有人都能看到成绩分析
        performance: false,
        export: false,
        settings: false,
        subjects: false,
        teacherAssignment: false,
        admin: false
    };
    
    if (userRole === '超级管理员') {
        // 超级管理员看到所有菜单
        Object.keys(menus).forEach(key => menus[key] = true);
    } else if (userRole === '正班主任') {
        // 正班主任看到班级管理相关菜单
        menus.students = true;
        menus.grades = true;
        menus.deyu = true;
        menus.comments = true;
        menus.export = true;
    } else if (['副班主任', '科任老师'].includes(userRole)) {
        // 副班主任和科任老师只看到成绩管理
        menus.grades = true;
    } else if (['行政', '校级领导'].includes(userRole)) {
        // 行政和校级领导看到成绩管理和班主任考核
        menus.grades = true;
        menus.performance = true;
    }
    
    return menus;
}
```

### 数据访问逻辑

#### 学生管理
```python
def can_access_students(user_role, user_class_id, target_class_id):
    if user_role == '超级管理员':
        return True
    if user_role == '正班主任' and user_class_id == target_class_id:
        return True
    return False
```

#### 成绩管理
```python
def can_edit_grade(user_role, user_class_id, user_subjects, target_class_id, target_subject):
    # 超级管理员可以编辑所有成绩
    if user_role == '超级管理员':
        return True
    
    # 正班主任可以编辑自己班级的所有成绩
    if user_role == '正班主任' and user_class_id == target_class_id:
        return True
    
    # 其他角色只能编辑自己任教学科的成绩
    if target_subject in user_subjects:
        return True
    
    return False
```

#### 班主任考核
```python
def can_access_performance(user_role):
    return user_role in ['超级管理员', '行政', '校级领导']
```

## 实现步骤

### 第一步：后端权限验证
1. 创建权限检查装饰器
2. 在每个API路由添加权限验证
3. 根据用户角色过滤数据

### 第二步：前端菜单控制
1. 根据用户角色动态显示/隐藏菜单
2. 在index.html中添加权限判断逻辑

### 第三步：数据过滤
1. 学生列表：只显示有权限的班级
2. 成绩管理：只显示可编辑的学科
3. 评语管理：只显示自己班级

### 第四步：界面提示
1. 无权限时显示友好提示
2. 尝试访问无权限页面时重定向

## 权限检查流程

```
用户登录
  ↓
获取用户信息（角色、班级、任教学科）
  ↓
前端：根据角色显示菜单
  ↓
用户点击菜单
  ↓
前端：检查是否有权限访问
  ↓
后端：API验证权限
  ↓
后端：根据权限过滤数据
  ↓
返回结果
```

## 错误处理

### 无权限访问
- HTTP 403 Forbidden
- 返回友好的错误消息
- 前端显示提示并重定向

### 数据越权
- 检查用户是否有权限访问该数据
- 记录越权尝试日志
- 返回403错误

## 安全考虑

1. **前后端双重验证**
   - 前端控制菜单显示
   - 后端验证API权限

2. **数据过滤**
   - 查询时自动添加权限过滤条件
   - 防止通过修改请求参数越权

3. **日志记录**
   - 记录所有权限检查
   - 记录越权尝试

4. **会话管理**
   - 定期刷新用户权限信息
   - 角色变更后立即生效
