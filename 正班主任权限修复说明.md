# 正班主任成绩管理权限修复说明

## 修复日期
2026-01-17

## 问题描述

### 发现的问题
正班主任在成绩管理页面中，只能编辑自己任教的学科，而不能编辑自己班级的所有学科成绩。这与设计的权限规则不符。

**预期行为：**
- 正班主任应该可以编辑自己班级的**所有学科**成绩

**实际行为：**
- 正班主任只能编辑自己任教的学科成绩
- 其他学科的下拉框被禁用（灰色显示）

### 问题原因

在 `utils/permission_checker.py` 的 `get_editable_subjects()` 函数中，判断正班主任权限时有一个额外的条件：

```python
# 错误的代码
if is_head_teacher(user) and class_id and str(user.class_id) == str(class_id):
    # 返回所有学科
```

这个条件要求：
1. 用户是正班主任
2. 必须提供 `class_id` 参数
3. `class_id` 必须等于用户的班级ID

**问题在于：**
- 前端调用 `get_editable_subjects()` 时没有传递 `class_id` 参数
- 即使传递了，在成绩管理页面中，正班主任只能看到自己的班级，所以不需要这个检查

## 修复方案

### 修改内容

**文件：** `utils/permission_checker.py`

**修改前：**
```python
# 正班主任可以编辑自己班级的所有学科
if is_head_teacher(user) and class_id and str(user.class_id) == str(class_id):
    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        cursor.execute('SELECT name FROM subjects')
        subjects = cursor.fetchall()
        conn.close()
        return [s['name'] for s in subjects]
    except Exception as e:
        logger.error(f"获取所有学科失败: {str(e)}")
        return []
```

**修改后：**
```python
# 正班主任可以编辑自己班级的所有学科
# 注意：正班主任在成绩管理页面只能看到自己的班级，所以这里不需要检查class_id
if is_head_teacher(user):
    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        cursor.execute('SELECT name FROM subjects')
        subjects = cursor.fetchall()
        conn.close()
        return [s['name'] for s in subjects]
    except Exception as e:
        logger.error(f"获取所有学科失败: {str(e)}")
        return []
```

### 修改说明

1. **移除了 `class_id` 检查**
   - 正班主任在成绩管理页面只能看到自己班级的学生
   - 不需要额外检查 `class_id` 参数
   - 简化了逻辑，避免了参数传递问题

2. **保持了安全性**
   - 后端的 `can_edit_grade()` 函数仍然会检查班级ID
   - 即使前端被绕过，后端仍然会验证权限
   - 双重验证机制保持不变

## 代码逻辑说明

### 权限检查流程

```
1. 前端加载成绩管理页面
   ↓
2. 调用 /api/current-user 获取权限信息
   ↓
3. 后端调用 get_user_permissions(user)
   ↓
4. get_user_permissions 调用 get_editable_subjects(user)
   ↓
5. get_editable_subjects 判断：
   - 如果是超级管理员 → 返回所有学科
   - 如果是正班主任 → 返回所有学科 ✅ (修复后)
   - 其他角色 → 返回任教学科
   ↓
6. 前端根据 editable_subjects 禁用/启用学科下拉框
   ↓
7. 用户修改成绩时，后端再次验证权限（双重验证）
```

### 双重验证机制

**前端验证（用户体验）：**
```javascript
// js/grades.js
function canEditSubject(subjectFieldName) {
    if (!userPermissions) return false;
    
    // 超级管理员可以编辑所有学科
    if (userPermissions.is_admin) return true;
    
    // 检查是否在可编辑学科列表中
    const editableSubjects = userPermissions.editable_subjects || [];
    return editableSubjects.includes(subjectDisplayName);
}
```

**后端验证（安全保障）：**
```python
# grades.py
def save_student_grade(student_id):
```

```python
# utils/permission_checker.py
def can_edit_grade(user, target_class_id, target_subject):
    # 超级管理员可以编辑所有成绩
    if is_super_admin(user):
        return True
    
    # 正班主任可以编辑自己班级的所有成绩
    if is_head_teacher(user) and str(user.class_id) == str(target_class_id):
        return True  # ✅ 不检查学科，直接返回True
    
    # 其他角色只能编辑自己任教学科的成绩
    user_subjects = get_user_subjects(user.id)
    subject_names = [s['name'] for s in user_subjects]
    return target_subject in subject_names
```

## 相关函数说明

### get_editable_subjects(user, class_id=None)
**作用：** 获取用户可以编辑的学科列表（用于前端显示）

**返回值：**
- 超级管理员：所有学科列表
- 正班主任：所有学科列表 ✅
- 其他角色：用户任教的学科列表

### can_edit_grade(user, target_class_id, target_subject)
**作用：** 检查用户是否可以编辑特定班级的特定学科成绩（用于后端验证）

**返回值：**
- 超级管理员：True（所有班级所有学科）
- 正班主任：True（如果是自己班级，不检查学科）✅
- 其他角色：True（如果学科在任教列表中）
