# 教师导入功能测试指南

## 测试环境
- 服务器状态：✅ 运行中（PID: 25547）
- 访问地址：http://localhost:5000

## 测试步骤

### 1. 访问管理后台
1. 打开浏览器访问：http://localhost:5000
2. 使用超级管理员账号登录
3. 点击右上角"管理后台"进入用户管理页面

### 2. 测试Excel模板下载
1. 点击"Excel导入教师名单"按钮
2. 点击"下载Excel模板"链接
3. 验证模板包含以下列：
   - 用户名（必填）
   - 年级（可选）
   - 班级（可选）
   - 角色（可选）
   - 任教学科（可选）

### 3. 测试预览功能

#### 测试用例1：新增用户（绿色）
创建测试Excel文件，包含以下数据：
```
用户名    年级      班级        角色        任教学科
测试1     一年级    一年级1班   正班主任    语文,数学
测试2     二年级              科任老师    英语
```

预期结果：
- 两条记录都显示为绿色（新增）
- 状态显示"新增"
- 统计信息：总数2，新增2，更新0，无效0

#### 测试用例2：更新用户（红色）
使用已存在的用户名：
```
用户名    年级      班级        角色        任教学科
admin     一年级    一年级1班   超级管理员  语文
```

预期结果：
- 记录显示为红色（更新）
- 状态显示"更新"
- 统计信息：总数1，新增0，更新1，无效0

#### 测试用例3：无效数据（灰色）
```
用户名    年级      班级          角色        任教学科
测试3     一年级    不存在的班级  正班主任    语文
测试4     二年级    二年级1班     无效角色    英语
测试5     三年级    三年级1班     科任老师    不存在的学科
```

预期结果：
- 三条记录都显示为灰色（无效）
- 状态显示"无效"
- 原因列显示具体错误信息
- 统计信息：总数3，新增0，更新0，无效3

#### 测试用例4：混合数据
```
用户名    年级      班级        角色        任教学科
新用户1   一年级    一年级1班   正班主任    语文,数学
admin     二年级    二年级1班   超级管理员  英语
新用户2   三年级              科任老师    体育
错误1     四年级    不存在班级  正班主任    语文
```

预期结果：
- 新用户1：绿色（新增）
- admin：红色（更新）
- 新用户2：绿色（新增）
- 错误1：灰色（无效）
- 统计信息：总数4，新增2，更新1，无效1

### 4. 测试确认导入

#### 测试新增用户
1. 上传包含新用户的Excel文件
2. 预览确认显示为绿色
3. 点击"确认导入"
4. 验证结果模态框：
   - 显示"新增账户密码信息"区域
   - 密码表格包含用户名、班级、密码
   - 密码为6位随机字符
5. 点击"复制所有密码信息"按钮
6. 验证剪贴板内容格式正确
7. 关闭模态框，验证用户列表已更新

#### 测试更新用户
1. 上传包含已存在用户的Excel文件
2. 预览确认显示为红色
3. 点击"确认导入"
4. 验证结果模态框：
   - 显示"更新的教师账户"区域
   - 更新表格包含用户名、班级、角色、任教学科
   - 不显示密码信息（因为是更新）
5. 关闭模态框，验证用户信息已更新

#### 测试混合导入
1. 上传包含新增、更新、无效数据的Excel文件
2. 预览确认颜色标记正确
3. 点击"确认导入"
4. 验证结果模态框：
   - 显示新增用户密码
   - 显示更新用户信息
   - 显示跳过记录及原因
5. 验证统计数字正确

### 5. 测试边界情况

#### 空用户名
```
用户名    年级      班级        角色
          一年级    一年级1班   正班主任
```
预期：显示为无效，原因"用户名为空"

#### 仅用户名
```
用户名
测试用户
```
预期：显示为新增（绿色），其他字段为空

#### 多个学科
```
用户名    任教学科
测试      语文,数学,英语,体育,音乐
```
预期：如果所有学科都存在，显示为新增；否则显示为无效

### 6. 测试实时更新
1. 导入成功后，不要手动刷新页面
2. 验证用户列表自动更新
3. 验证新增/更新的用户显示在列表中

## 常见问题

### Q1: 预览时显示"班级不存在"
A: 需要先在"班级管理"选项卡中创建对应的班级

### Q2: 预览时显示"学科不存在"
A: 需要先在"学科管理"页面中创建对应的学科

### Q3: 更新用户后密码是否改变？
A: 不会。更新用户只修改角色、班级、任教学科，不修改密码

### Q4: 如何重置已导入用户的密码？
A: 在用户列表中点击"重置密码"按钮

### Q5: 导入后如何查看用户的当前密码？
A: 在用户列表的"当前密码"列可以看到
