# 任教学科删除同步问题 - 已修复

## 问题描述

用户在后台删除教师的任教学科后，发现：
1. **学科班级分配的数据还在**（书法、道法还显示在任教信息中）
2. **成绩管理里面还能编辑**这些已删除的学科

## 问题原因

系统使用了两个表来管理教师的任教信息：

### 1. `teacher_subjects` 表
- 记录教师任教哪些学科
- 结构：`teacher_id` + `subject_id`
- 用途：在教师分配页面显示"任教学科"

### 2. `teaching_assignments` 表
- 记录教师在哪些班级教哪些学科
- 结构：`teacher_id` + `class_id` + `subject`（学科名称）
- 用途：
  - 在教师分配页面显示"学科班级分配"
  - 在成绩管理中控制编辑权限
  - 在首页显示科任老师的任教信息和待办事项

### 问题所在

在 `subjects.py` 的 `remove_subject_from_teacher()` 函数中：
- ✅ 删除了 `teacher_subjects` 表的记录
- ❌ **没有删除** `teaching_assignments` 表的记录

这导致：
- 任教学科列表中看不到该学科了（因为 `teacher_subjects` 记录被删除）
- 但学科班级分配还在（因为 `teaching_assignments` 记录还在）
- 成绩管理的权限检查基于 `teaching_assignments` 表，所以还能编辑

## 修复方案

### 1. 修改删除逻辑（已完成）

修改 `subjects.py` 中的 `remove_subject_from_teacher()` 函数：

```python
@subjects_bp.route('/api/teacher-subjects/<teacher_id>/<int:subject_id>', methods=['DELETE'])
@login_required
def remove_subject_from_teacher(teacher_id, subject_id):
    """移除教师的任教学科"""
    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        
        # 获取学科名称
        cursor.execute('SELECT name FROM subjects WHERE id = ?', (subject_id,))
        subject_row = cursor.fetchone()
        subject_name = subject_row['name'] if subject_row else None
        
        # 1. 移除任教学科（teacher_subjects 表）
        cursor.execute('''
            DELETE FROM teacher_subjects 
            WHERE teacher_id = ? AND subject_id = ?
        ''', (teacher_id, subject_id))
        
        # 2. 同步删除学科班级分配（teaching_assignments 表）⭐ 新增
        if subject_name:
            cursor.execute('''
                DELETE FROM teaching_assignments 
                WHERE teacher_id = ? AND subject = ?
            ''', (teacher_id, subject_name))
            deleted_assignments = cursor.rowcount
            logger.info(f"同步删除了 {deleted_assignments} 条学科班级分配记录")
        
        conn.commit()
        conn.close()
        
        return jsonify({
            'status': 'ok',
            'message': '学科移除成功，相关班级分配也已清除'
        })
        
    except Exception as e:
        logger.error(f"移除学科时出错: {e}")
        if conn:
            conn.rollback()
        return jsonify({
            'status': 'error',
            'message': f'移除学科失败: {str(e)}'
        }), 500
```

## 相关表结构

### teacher_subjects 表
```sql
CREATE TABLE teacher_subjects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    teacher_id TEXT NOT NULL,
    subject_id INTEGER NOT NULL,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (teacher_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (subject_id) REFERENCES subjects(id) ON DELETE CASCADE,
    UNIQUE(teacher_id, subject_id)
)
```

### teaching_assignments 表
```sql
CREATE TABLE teaching_assignments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    teacher_id TEXT NOT NULL,
    class_id TEXT NOT NULL,
    subject TEXT NOT NULL,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(teacher_id, class_id, subject)
)
```

## 权限检查逻辑

成绩管理的权限检查（`utils/permission_checker.py`）：

```python
def can_edit_grade(user, target_class_id, target_subject):
    """检查是否有权限编辑成绩"""
    # 超级管理员可以编辑所有成绩
    if is_super_admin(user):
        return True
    
    # 正班主任可以编辑自己班级的所有成绩
    if is_head_teacher(user) and str(user.class_id) == str(target_class_id):
        return True
    
    # 其他角色：必须精确匹配班级和学科（从 teaching_assignments 表）⭐
    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        cursor.execute('''
            SELECT COUNT(*) as count
            FROM teaching_assignments
            WHERE teacher_id = ? AND class_id = ? AND subject = ?
        ''', (user.id, str(target_class_id), target_subject))
        result = cursor.fetchone()
        conn.close()
        return result['count'] > 0
    except Exception as e:
        logger.error(f"检查编辑权限失败: {str(e)}")
        return False
```

## 预防措施

### 1. 数据库外键约束

考虑在 `teaching_assignments` 表中添加外键约束，但由于 `subject` 字段存储的是学科名称（TEXT），而不是学科ID，所以无法直接使用外键。

### 2. 级联删除

如果将来重构表结构，建议：
- `teaching_assignments` 表的 `subject` 字段改为 `subject_id`（存储学科ID）
- 添加外键约束：`FOREIGN KEY (subject_id) REFERENCES subjects(id) ON DELETE CASCADE`
- 这样删除学科时会自动级联删除相关记录

### 3. 定期检查

可以定期运行清理脚本，检查是否有新的孤立记录：
