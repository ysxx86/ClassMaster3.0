### 方案3：数据库优化

#### 3.1 分离评语表（推荐）

**当前问题**：
- students表包含评语，导致表臃肿
- 每次查询都要传输大量评语数据

**解决方案**：
```sql
-- 创建独立的评语表
CREATE TABLE student_comments (
    student_id TEXT NOT NULL,
    class_id TEXT NOT NULL,
    content TEXT,
    updated_at TEXT,
    PRIMARY KEY (student_id, class_id),
    FOREIGN KEY (student_id, class_id) REFERENCES students(id, class_id)
);

-- 创建索引
CREATE INDEX idx_comments_class ON student_comments(class_id);
CREATE INDEX idx_comments_updated ON student_comments(updated_at);

-- 迁移数据
INSERT INTO student_comments (student_id, class_id, content, updated_at)
SELECT id, class_id, comments, updated_at 
FROM students 
WHERE comments IS NOT NULL AND comments != '';

-- 清空students表的comments字段
UPDATE students SET comments = NULL;
```

**优势**：
- students表变小，查询更快
- 评语按需加载
- 可以独立优化评语查询

```javascript
// 分批渲染，每批50个
const BATCH_SIZE = 50;
let currentBatch = 0;

function renderBatch() {
    const start = currentBatch * BATCH_SIZE;
    const end = Math.min(start + BATCH_SIZE, allStudents.length);
    
    for (let i = start; i < end; i++) {
        const card = createCommentCard(allStudents[i]);
        commentCards.appendChild(card);
    }
    
    currentBatch++;
    
    if (end < allStudents.length) {
        // 继续渲染下一批
        requestAnimationFrame(renderBatch);
    } else {
        console.log('渲染完成');
    }
}

// 开始渲染
renderBatch();
```
