### 2. 科任老师只显示任教学科（TASK 12）
**功能**：科任老师访问成绩管理时，只显示自己任教的学科列，不显示其他学科

**优势**：
1. **保护学生隐私** - 其他学科成绩不可见
2. **界面更简洁** - 只显示相关信息
3. **提高工作效率** - 不被无关信息干扰

**实现逻辑**：
- **超级管理员**：显示所有学科列
- **正班主任**：显示所有学科列（可以管理本班级所有学科）
- **科任老师/副班主任/行政/校级领导**：只显示自己任教的学科列

**修改文件**：

#### 后端修改（`grades.py`）
1. 修改 `/api/grades` API：
   - 导入 `get_user_permissions` 函数
   - 获取用户完整权限信息（包含 `teaching_map`）
   - 返回 `user_permissions` 和 `allowed_subjects`（允许查看的学科列表）
   - 对于多班级用户，合并所有任教班级的学科列表

#### 前端修改（`js/grades.js`）
1. 添加全局变量 `allowedSubjects` 存储允许查看的学科列表

2. 修改 `loadGrades()` 函数：
   - 保存 API 返回的 `user_permissions` 和 `allowed_subjects`
   - 传递给 `renderGradesTable()` 函数

3. 修改 `renderGradesTable()` 函数：
   - 根据用户角色确定要渲染的学科列表
   - 超级管理员和正班主任：显示所有学科
   - 其他角色：只显示任教的学科
   - 动态更新表头，只显示要渲染的学科列
   - 只渲染要显示的学科单元格
   - 调整 `colspan` 以适应实际列数

### 数据流程

2. 后端查询用户权限和任教信息
3. 后端返回成绩数据 + 权限信息 + 允许查看的学科列表
4. 前端根据权限信息动态生成表头和表格
5. 前端根据权限控制编辑功能
