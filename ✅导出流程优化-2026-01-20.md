## 🔴 问题 1：班级名称查询错误（已修复）

### 问题描述
```
2026-01-20 11:22:13,363 - ERROR - 获取班级名称时出错: no such column: name
```

**原因：**
- 代码使用了错误的列名 `name`
- `classes` 表的实际列名是 `class_name`

**影响：**
- 合并 PDF 文件名变成 `合并报告.pdf` 而不是 `五年级1班_合并报告.pdf`
- 多个班级导出时容易混淆

### 修复方案
```python
# 修改前
cursor.execute('SELECT name FROM classes WHERE id = ?', (class_id,))
if class_record and 'name' in class_record:
    class_name = class_record['name']

# 修改后
cursor.execute('SELECT class_name FROM classes WHERE id = ?', (class_id,))
if class_record and 'class_name' in class_record:
    class_name = class_record['class_name']
```

**修复文件：** `comments.py` 第 1806 行

## 🟡 问题 2：运行时安装依赖（需注意）

### 问题描述
```
2026-01-20 11:22:11,267 - ERROR - 无法导入PyPDF2库，尝试安装
Collecting PyPDF2
Successfully installed PyPDF2-3.0.1
```

**原因：**
- 虚拟环境中缺少 `PyPDF2` 库
- 系统在运行时自动安装（增加 1-2 秒延迟）

**影响：**
- 用户体验：导出过程中突然暂停
- 不符合生产环境最佳实践

### 解决方案
确保虚拟环境已安装所有依赖：

```bash
# 激活虚拟环境（PowerShell）
.\venv\Scripts\Activate.ps1

# 安装所有依赖
pip install -r requirements.txt

# 验证 PyPDF2 是否已安装
pip show PyPDF2
```

**注意：** `PyPDF2` 已在 `requirements.txt` 中，只需重新安装依赖即可。

## 🟢 问题 3：Word 退出警告（已优化）

### 问题描述
```
2026-01-20 11:22:09,324 - WARNING - 退出Word时出错
2026-01-20 11:22:11,261 - WARNING - 退出Word时出错
```

**原因：**
- Word COM 对象退出时的正常行为
- 不影响功能，文件已成功转换

**影响：**
- 日志略显混乱，可能误导用户

### 优化方案
将警告降级为 DEBUG 日志：

```python
# 修改前
except:
    logger.warning("退出Word时出错")

# 修改后
except Exception as e:
    logger.debug(f"退出Word时出错（可忽略）: {str(e)}")
```

**修复文件：** `comments.py` 第 1077 行
