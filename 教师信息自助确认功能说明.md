# 教师信息自助确认功能说明

## 📋 功能概述

教师信息自助确认功能允许教师在登录时查看和确认自己的任教信息，包括任教学科和班级分配。这是对现有管理员分配功能的补充和增强。

## ✨ 主要特性

### 1. 双轨制管理
- **管理员分配**：管理员可以在后台批量分配教师的角色、班级和学科
- **教师自助**：教师登录后可以查看、确认和修改自己的任教信息
- **数据同步**：两种方式的数据实时同步，互不冲突

### 2. 自动弹窗提醒
教师登录时，系统会自动检测是否需要确认信息，以下情况会弹出确认窗口：
- 首次登录系统
- 新学年/新学期开始
- 距离上次确认超过90天
- 信息未填写完整

### 3. 信息展示
弹窗中会显示：
- 基本信息（姓名、角色、班级）
- 任教学科列表
- 学科班级分配详情

### 4. 灵活修改
教师可以：
- 添加或删除任教学科
- 为每个学科选择任教的班级
- 随时保存修改

### 5. 历史记录
系统会记录：
- 每次确认的时间
- 每次修改的内容
- 修改前后的对比

## 🚀 使用流程

### 教师端流程

1. **登录系统**
   - 输入用户名和密码登录

2. **查看确认弹窗**
   - 系统自动弹出任教信息确认窗口
   - 显示当前学期和任教信息

3. **确认或修改**
   - **信息正确**：点击"确认无误"按钮
   - **信息有误**：点击各部分的"修改"按钮进行编辑

4. **编辑任教学科**
   - 点击"任教学科"旁的"修改"按钮
   - 勾选或取消勾选学科
   - 点击"保存"按钮

5. **编辑学科班级分配**
   - 点击"学科班级分配"旁的"修改"按钮
   - 为每个学科选择任教的班级
   - 点击"保存"按钮

6. **最终确认**
   - 检查所有信息无误后
   - 点击"确认无误"按钮
   - 系统保存并关闭弹窗

### 管理员端流程

1. **批量分配**（可选）
   - 在"教师分配"页面进行初始分配
   - 作为教师确认的参考数据

2. **查看统计**
   - 查看有多少教师已确认信息
   - 查看未确认教师列表

3. **审核修改**（可选）
   - 查看教师的修改历史
   - 必要时进行调整

## 🔧 安装步骤

### 步骤1：运行数据库迁移

```bash
python migrate_add_teacher_confirmation.py
```

这会：
- 在users表添加确认相关字段
- 创建teacher_info_history历史记录表
- 初始化现有教师的学期信息

### 步骤2：注册API蓝图

在 `server.py` 或 `app.py` 中添加：

```python
from teacher_info_api import teacher_info_bp

# 注册教师信息API蓝图
app.register_blueprint(teacher_info_bp)
```

### 步骤3：在页面中引入文件

在需要显示确认弹窗的页面（如首页）的 `<head>` 中添加：

```html
<!-- 教师信息确认样式 -->
<link rel="stylesheet" href="/css/teacher-info-confirmation.css">
```

在页面底部 `</body>` 前添加：

```html
<!-- 教师信息确认脚本 -->
<script src="/js/teacher-info-confirmation.js"></script>
```

### 步骤4：重启服务器

```bash
# 停止服务器
pkill -f "python.*server.py"

# 启动服务器
python server.py
```

## 📊 数据库结构

### users表新增字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| info_confirmed | INTEGER | 是否已确认（0=未确认，1=已确认） |
| last_confirmed_date | TEXT | 最后确认日期 |
| current_semester | TEXT | 当前学期 |

### teacher_info_history表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键 |
| teacher_id | TEXT | 教师ID |
| action | TEXT | 操作描述 |
| action_type | TEXT | 操作类型（confirm/update） |
| old_data | TEXT | 修改前数据（JSON） |
| new_data | TEXT | 修改后数据（JSON） |
| ip_address | TEXT | IP地址 |
| user_agent | TEXT | 浏览器信息 |
| created_at | TEXT | 创建时间 |

## 🔌 API接口

### 1. 检查是否需要确认
```
GET /api/teacher-info/check-confirmation
```

**返回：**
```json
{
    "status": "ok",
    "need_confirm": true,
    "reason": "never_confirmed",
    "current_semester": "2025-2026学年第二学期"
}
```

### 2. 获取教师信息
```
GET /api/teacher-info/my-info
```

**返回：**
```json
{
    "status": "ok",
    "info": {
        "username": "张老师",
        "role": "科任老师",
        "subjects": [...],
        "assignments": [...]
    }
}
```

### 3. 确认信息
```
POST /api/teacher-info/confirm
```

### 4. 更新信息
```
POST /api/teacher-info/update
```

**请求体：**
```json
{
    "subjects": [1, 2, 3],
    "assignments": [
        {"class_id": 1, "subject_name": "语文"},
        {"class_id": 2, "subject_name": "数学"}
    ]
}
```

### 5. 获取修改历史
```
GET /api/teacher-info/history
```

### 6. 管理员统计（仅管理员）
```
GET /api/teacher-info/admin/stats
```

## 🎯 与现有功能的关系

### 与管理员分配功能
- **互补关系**：管理员分配提供初始数据，教师自助提供确认和修正
- **数据同步**：两者操作同一数据表，实时同步
- **权限分离**：管理员可以修改所有人，教师只能修改自己

### 与权限系统
- **权限基础**：教师确认的信息用于权限判断
- **精确控制**：确保教师只能访问自己任教的班级和学科
- **动态更新**：教师修改后权限立即生效

### 与成绩管理
- **数据来源**：成绩管理的权限基于教师确认的信息
- **班级过滤**：只显示教师任教的班级
- **学科过滤**：只能编辑任教的学科

## 📈 统计功能

管理员可以查看：
- 教师总数
- 已确认人数
- 未确认人数
- 未确认教师列表

## 🔒 安全性

1. **权限验证**：所有API都需要登录
2. **数据隔离**：教师只能访问自己的数据
3. **操作记录**：所有修改都有历史记录
4. **IP记录**：记录操作的IP地址

## 🐛 常见问题

### Q1：弹窗不显示？
**A**：检查：
- 是否已登录
- 是否是超级管理员（管理员不显示）
- 浏览器控制台是否有错误
- 文件是否正确引入

### Q2：修改后不生效？
**A**：
- 检查网络请求是否成功
- 查看浏览器控制台错误
- 确认数据库连接正常

### Q3：如何重置确认状态？
**A**：在数据库中执行：
```sql
UPDATE users SET info_confirmed = 0 WHERE id = '教师ID';
```

### Q4：如何查看修改历史？
**A**：
- 教师端：调用 `/api/teacher-info/history`
- 数据库：查询 `teacher_info_history` 表
