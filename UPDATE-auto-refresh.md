# 绩效考核系统 - 自动刷新功能

## 更新内容

### 新增功能：自动刷新

绩效考核页面现在支持**自动刷新**，当后台管理员修改用户角色后，评分页面会自动更新教师列表，无需手动刷新浏览器。

## 功能特性

### 1. 自动定时刷新
- **刷新频率**: 每30秒自动检查一次数据
- **智能检测**: 只有当数据发生变化时才更新界面
- **后台运行**: 在评分录入页面时自动运行

### 2. 手动刷新按钮
- 位置：评分录入页面右上角
- 功能：立即刷新数据，无需等待30秒
- 图标：🔄 刷新按钮

### 3. 智能暂停/恢复
- **切换标签页时**: 自动停止刷新（节省资源）
- **返回评分页面时**: 自动恢复刷新
- **页面失去焦点时**: 暂停刷新
- **页面获得焦点时**: 恢复刷新并立即刷新一次

### 4. 数据变化检测
- 只有当教师数量或教师列表发生变化时才更新界面
- 避免不必要的界面刷新
- 保持用户当前的操作状态

### 5. 友好提示
- 自动刷新时显示提示："数据已自动更新"
- 手动刷新时显示提示："数据已刷新"
- 提示3秒后自动消失

## 使用场景

### 场景1：管理员修改用户角色
1. 管理员在后台管理页面修改某个用户的角色（如：从"正班主任"改为"副班主任"）
2. 保存修改
3. **30秒内**，绩效考核页面会自动更新，该用户从列表中消失
4. 无需手动刷新浏览器

### 场景2：添加新的正班主任
1. 管理员在后台添加新用户，角色设为"正班主任"
2. 保存
3. **30秒内**，绩效考核页面会自动更新，新教师出现在列表中

### 场景3：需要立即查看更新
1. 管理员修改了用户角色
2. 不想等待30秒
3. 点击右上角的"🔄 刷新"按钮
4. 数据立即更新

## 技术实现

### 前端实现
**文件**: `js/performance.js?v=20260117-2`

#### 核心功能：

1. **定时器管理**
```javascript
// 启动自动刷新（每30秒）
function startAutoRefresh() {
    autoRefreshInterval = setInterval(function() {
        loadScoreMatrixSilent();
    }, 30000);
}

// 停止自动刷新
function stopAutoRefresh() {
    clearInterval(autoRefreshInterval);
}
```

2. **静默刷新**

```javascript
// 计算数据哈希
const newDataHash = JSON.stringify({
    teacherCount: response.teachers.length,
    teacherIds: response.teachers.map(t => t.id).sort().join(',')
```

4. **页面焦点管理**


## 配置选项

### 修改刷新频率
如果需要修改刷新频率，编辑 `js/performance.js` 文件：

```javascript
// 找到这一行（约第60行）
}, 30000); // 30秒

// 修改为你想要的时间（毫秒）
}, 60000); // 60秒（1分钟）
}, 15000); // 15秒
}, 10000); // 10秒
```

**建议值**:
- 10秒：频繁更新，适合测试
- 30秒：平衡性能和实时性（默认）
- 60秒：节省资源，适合稳定环境

### 禁用自动刷新
如果不需要自动刷新功能，注释掉启动代码：

```javascript
// 在 $(document).ready 函数中找到这一行
// startAutoRefresh(); // 注释掉这一行
```

## 性能优化

### 1. 智能检测
- 只在数据真正变化时更新界面
- 避免不必要的DOM操作

### 2. 资源节省
- 切换到其他标签页时自动停止刷新
- 页面不可见时不消耗资源

### 3. 网络优化
- 使用轻量级的数据哈希比较
- 避免频繁的完整数据传输

### 4. 用户体验
- 静默刷新，不打断用户操作
- 只在必要时显示提示消息
- 提示自动消失，不影响界面

## 故障排除

### 问题1：自动刷新不工作
**解决方案**:
1. 清除浏览器缓存（Ctrl+Shift+R）
2. 检查浏览器控制台是否有错误
3. 确认服务器正在运行
4. 检查网络连接

### 问题2：刷新太频繁
**解决方案**:
1. 修改刷新频率（见"配置选项"）
2. 或禁用自动刷新，只使用手动刷新

### 问题3：数据不更新
**解决方案**:
1. 检查后台是否真的修改了数据
2. 运行测试脚本验证：`python3 test_performance_api.py`
3. 检查API响应：浏览器Network标签
4. 手动点击刷新按钮

### 问题4：提示消息不显示
**解决方案**:
1. 检查Bootstrap是否正确加载
2. 检查浏览器控制台是否有错误
3. 提示消息3秒后自动消失，可能错过了
