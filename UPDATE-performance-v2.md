# 班主任绩效考核系统 - V2.0 更新说明

## 更新日期
2026-01-17

## 主要更新

### 1. 角色系统重构 ✨

#### 旧版本
- 只有"班主任"一个角色
- 使用`teacher_type`字段区分正班主任和副班主任

#### 新版本
- **6种角色**：
  1. 正班主任
  2. 副班主任
  3. 科任老师
  4. 行政
  5. 校级领导
  6. 超级管理员

- 使用`role`字段统一管理角色
- 自动迁移旧数据（teacher_type → role）

### 2. 评分界面重构 🎯

#### 旧版本
- 下拉选择单个教师
- 纵向列表显示考核项目
- 逐个项目评分

#### 新版本
- **矩阵式评分界面**
- 横向显示所有考核项目
- 纵向显示所有正班主任
- 一次性查看所有评分情况
- 支持快速编辑和保存

### 3. 界面优化 💎

#### 评分矩阵特性
- 第一行：考核类别（合并单元格）
- 第二行：具体项目名称和权重
- 每个单元格显示：
  - 已有评分（多人评分显示多个）
  - 编辑按钮
  - 输入框（点击编辑时显示）

#### 固定列设计
- 第一列（教师姓名）固定
- 横向滚动查看所有项目
- 提升大屏幕使用体验

### 4. 数据库更新 📊

#### 新增字段
```sql
ALTER TABLE users ADD COLUMN role TEXT DEFAULT '科任老师';
```

#### 数据迁移
```sql
UPDATE users SET role = COALESCE(teacher_type, '科任老师');
```

### 5. API更新 🔧

#### 新增接口
- `GET /api/performance/scores/<semester>` - 获取评分矩阵数据
  - 返回所有正班主任列表
  - 返回所有考核项目
  - 返回所有评分记录

#### 修改接口
- `GET /api/performance/teachers` - 返回role字段
- `PUT /api/performance/teachers/<id>/role` - 更新角色（原type改为role）
- `POST /api/performance/calculate/<semester>` - 只计算正班主任

## 使用说明

### 评分流程（新版）

1. **进入评分界面**
   - 点击"评分录入"标签
   - 选择学期

2. **查看评分矩阵**
   - 横向：所有考核项目（按类别分组）
   - 纵向：所有正班主任
   - 单元格：显示已有评分

3. **录入评分**
   - 点击单元格中的"编辑"按钮
   - 输入分数（0-100）
   - 点击"保存"按钮（或再次点击编辑按钮）

4. **查看结果**
   - 切换到"考核结果"标签
   - 管理员点击"计算结果"
   - 查看排名和总分

### 教师角色管理

1. **进入教师管理**
   - 点击"教师管理"标签（管理员）

2. **设置角色**
   - 为每位教师选择角色：
     - 正班主任：参与考核
     - 副班主任：不参与考核
     - 科任老师：普通教师
     - 行政：行政人员
     - 校级领导：学校领导
     - 超级管理员：系统管理员

3. **保存设置**
   - 点击"保存"按钮

    }
}
