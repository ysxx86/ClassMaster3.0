# 权限系统测试指南

## 测试准备

### 1. 创建测试账号

需要创建以下角色的测试账号:

```sql
-- 1. 超级管理员 (已存在)
-- 用户名: admin
-- 密码: admin123

-- 2. 正班主任
-- 用户名: teacher_head
-- 角色: 正班主任
-- 班级: 选择一个班级

-- 3. 副班主任
-- 用户名: teacher_vice
-- 角色: 副班主任
-- 任教学科: 选择1-2个学科

-- 4. 科任老师
-- 用户名: teacher_subject
-- 角色: 科任老师
-- 任教学科: 选择1-2个学科

-- 5. 行政
-- 用户名: teacher_admin
-- 角色: 行政
-- 任教学科: 可选

-- 6. 校级领导
-- 用户名: teacher_leader
-- 角色: 校级领导
-- 任教学科: 可选
```

### 2. 重启服务器

修改代码后需要重启服务器:
```bash
# 停止当前服务器 (Ctrl+C)
# 重新启动
python server.py
```

## 常见问题排查

### 问题1: 菜单没有正确显示/隐藏

**可能原因**:
- 前端JavaScript没有正确执行
- /api/current-user API没有返回权限信息
- 浏览器缓存问题

**解决方法**:
1. 打开浏览器开发者工具(F12)
2. 查看Console标签,检查是否有JavaScript错误
3. 查看Network标签,检查/api/current-user的响应
4. 清除浏览器缓存并刷新页面(Ctrl+Shift+R)

### 问题2: API返回403错误

**可能原因**:
- 权限装饰器没有正确应用
- 用户角色设置不正确
- Session过期

**解决方法**:
1. 检查用户的primary_role字段是否正确设置
2. 重新登录获取新的session
3. 检查后端日志,查看权限检查的详细信息

### 问题3: 权限检查工具导入失败

**可能原因**:
- utils/permission_checker.py文件不存在
- Python路径问题

**解决方法**:
1. 确认utils/permission_checker.py文件存在
2. 检查文件权限
3. 重启服务器
