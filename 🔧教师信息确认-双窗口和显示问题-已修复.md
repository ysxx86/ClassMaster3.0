# 教师信息确认 - 双窗口和显示问题 - 已修复

## 问题描述

用户反馈了两个问题：

### 问题1：学科班级分配显示"暂无"
- 在教师分配页面显示"英语: 六年级3班（自动）"
- 但在教师信息确认弹窗中显示"暂无学科班级分配"

### 问题2：弹出两个窗口
- 登录后需要点击两次"关闭"按钮
- 第一次关闭一个窗口，第二次才能关闭第二个窗口

## 问题原因

### 问题1原因：正班主任的学科班级分配逻辑

**数据存储方式：**
- 正班主任的学科班级分配是"自动"的，不存储在 `teaching_assignments` 表中
- 系统逻辑：正班主任的所有任教学科自动关联到其班级
- 但前端代码只检查 `teaching_assignments` 表的数据

**代码问题：**
```javascript
// 原代码
function renderAssignmentsDisplay() {
    if (!teacherInfo.assignments || teacherInfo.assignments.length === 0) {
        return '<p class="text-muted">暂无学科班级分配</p>';  // ❌ 直接返回"暂无"
    }
    // ...
}
```

### 问题2原因：重复的事件监听器

**代码问题：**
在 `js/teacher-info-confirmation.js` 文件底部有两段重复的代码：

```javascript
// 第一段（第 476-482 行）
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', () => {
        setTimeout(checkTeacherInfoConfirmation, 1000);
    });
} else {
    setTimeout(checkTeacherInfoConfirmation, 1000);
}

// 第二段（第 598-607 行）
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', () => {
        setTimeout(() => {
            checkTeacherInfoConfirmation();  // ❌ 重复调用
            checkAndAddButton();
        }, 1000);
    });
} else {
    setTimeout(() => {
        checkTeacherInfoConfirmation();  // ❌ 重复调用
        checkAndAddButton();
    }, 1000);
}
```

这导致 `checkTeacherInfoConfirmation()` 被调用了两次，创建了两个模态框。

## 修复方案

### 修复1：正班主任的学科班级分配显示

修改 `renderAssignmentsDisplay()` 函数，增加正班主任的特殊处理：

```javascript
function renderAssignmentsDisplay() {
    // ⭐ 新增：如果是正班主任且有班级，自动显示所有学科对应该班级
    if (teacherInfo.role === '正班主任' && teacherInfo.class_name && teacherInfo.subjects && teacherInfo.subjects.length > 0) {
        let html = '<div class="table-responsive"><table class="table table-sm table-bordered">';
        html += '<thead><tr><th>学科</th><th>任教班级</th></tr></thead><tbody>';
        
        teacherInfo.subjects.forEach(subject => {
            html += `<tr><td><strong>${subject.name}</strong></td><td>${teacherInfo.class_name} <span class="badge bg-success">自动</span></td></tr>`;
        });
        
        html += '</tbody></table></div>';
        html += '<p class="text-muted small mt-2"><i class="bx bx-info-circle"></i> 正班主任的所有任教学科自动关联到本班级</p>';
        return html;
    }
    
    // 其他角色：从 teaching_assignments 获取
    if (!teacherInfo.assignments || teacherInfo.assignments.length === 0) {
        return '<p class="text-muted">暂无学科班级分配</p>';
    }
    
    // ... 原有代码
}
```

**效果：**
- 正班主任：显示所有任教学科 + 本班级 + "自动"标记
- 其他角色：显示 `teaching_assignments` 表中的数据

### 修复2：禁止正班主任编辑学科班级分配

因为正班主任的学科班级分配是自动的，不应该让他们手动编辑：

```javascript
// 在模态框HTML中，条件渲染"修改"按钮
<h6 class="section-title">
    学科班级分配
    ${teacherInfo.role !== '正班主任' ? `
    <button class="btn btn-sm btn-outline-primary float-end" onclick="editAssignments()">
        <i class='bx bx-edit'></i> 修改
    </button>
    ` : ''}
</h6>
```

### 修复3：删除重复的事件监听器

删除第一段重复的代码（第 476-482 行）：

```javascript
// ❌ 删除这段代码
// 页面加载完成后检查
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', () => {
        setTimeout(checkTeacherInfoConfirmation, 1000);
    });
} else {
    setTimeout(checkTeacherInfoConfirmation, 1000);
}
```

保留第二段（包含 `checkAndAddButton()`）：

```javascript
// ✅ 保留这段代码
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', () => {
        setTimeout(() => {
            checkTeacherInfoConfirmation();
            checkAndAddButton();
        }, 1000);
    });
} else {
    setTimeout(() => {
        checkTeacherInfoConfirmation();
        checkAndAddButton();
    }, 1000);
}
```

## 注意事项

### 1. 正班主任的数据一致性

正班主任的学科班级分配是"虚拟"的（不存储在数据库中），所以：
- 不需要在 `teaching_assignments` 表中创建记录
- 权限检查时，正班主任可以编辑本班级的所有学科
- 在"全部分配"页面显示时，会自动标记为"自动"

### 2. 保存修改时的处理

当正班主任保存修改时：
- 只更新 `teacher_subjects` 表（任教学科）
- 不更新 `teaching_assignments` 表（因为是自动的）

### 3. 权限检查逻辑

在 `utils/permission_checker.py` 中：
