# 成绩管理权限优化说明

## 优化日期
2026-01-17

## 优化内容

### 问题描述
之前的实现中，当用户尝试编辑没有权限的学科成绩时，前端允许修改，但后端返回403错误，在页面右侧显示小的错误提示。这种方式不够直观，用户可能以为修改成功了，但实际上失败了。

### 优化方案
**直接禁用不可编辑的学科输入框**，让用户一眼就能看出哪些学科可以编辑，哪些不能编辑。

### 实施细节

#### 1. 前端权限检查 (js/grades.js)

**添加权限获取函数**
```javascript
// 存储用户权限信息
let userPermissions = null;

// 获取用户权限信息
async function fetchUserPermissions() {
    try {
        const response = await fetch('/api/current-user');
        const data = await response.json();
        if (data.status === 'ok' && data.permissions) {
            userPermissions = data.permissions;
            console.log('用户权限信息:', userPermissions);
        }
    } catch (error) {
        console.error('获取用户权限失败:', error);
    }
}

// 检查是否可以编辑某个学科
function canEditSubject(subjectFieldName) {
    if (!userPermissions) {
        return false;
    }
    
    // 超级管理员可以编辑所有学科
    if (userPermissions.is_admin) {
        return true;
    }
    
    // 获取学科显示名称
    const subjectDisplayName = subjectNames[subjectFieldName];
    if (!subjectDisplayName) {
        return false;
    }
    
    // 检查是否在可编辑学科列表中
    const editableSubjects = userPermissions.editable_subjects || [];
    return editableSubjects.includes(subjectDisplayName);
}
```

**在页面加载时获取权限**
```javascript
document.addEventListener('DOMContentLoaded', function() {
    // 获取用户权限信息
    fetchUserPermissions().then(() => {
        // 初始化学期选择和数据
        setupSemesterSelect();
        // ... 其他初始化代码
    });
});
```

**渲染表格时禁用不可编辑的学科**
```javascript
// 添加各科目成绩单元格
subjectsToRender.forEach(subject => {
    const cell = document.createElement('td');
    const gradeValue = studentGrade[subject] !== undefined ? studentGrade[subject] : '';
    
    // 检查是否可以编辑该学科
    const canEdit = canEditSubject(subject);
    
    // 创建成绩选择框
    const select = document.createElement('select');
    select.className = 'form-select form-select-sm grade-select';
    select.setAttribute('data-student-id', studentGrade.student_id);
    select.setAttribute('data-subject', subject);
    
    // 如果没有编辑权限，禁用选择框并添加提示
    if (!canEdit) {
        select.disabled = true;
        select.classList.add('disabled-subject');
        select.title = '您没有权限编辑此学科';
    }
    
    // ... 添加选项和样式
});
```

### 用户体验改进

#### 优化前
1. 用户可以点击任何学科的下拉框
2. 用户可以选择成绩等级
3. 系统尝试保存
4. 后端返回403错误
5. 页面右侧显示小的错误提示
6. 用户可能没注意到错误提示，以为保存成功了

#### 优化后
1. 页面加载时自动获取用户权限
2. 不可编辑的学科下拉框直接禁用（灰色显示）
3. 鼠标悬停时显示"您没有权限编辑此学科"
4. 用户无法点击或修改禁用的学科
5. 只有可编辑的学科才能修改
6. 一目了然，避免误操作

### 权限规则

| 角色 | 可编辑学科 |
|------|-----------|
| 超级管理员 | 所有学科 |
| 正班主任 | 自己班级的所有学科 |
| 副班主任 | 只有自己任教的学科 |
| 科任老师 | 只有自己任教的学科 |
| 行政 | 只有自己任教的学科（如果有） |
| 校级领导 | 只有自己任教的学科（如果有） |

### 技术实现

#### 数据流程
```
1. 页面加载
   ↓
2. 调用 /api/current-user 获取权限信息
   ↓
3. 存储 userPermissions (包含 editable_subjects)
   ↓
4. 加载成绩数据
   ↓
5. 渲染表格时，对每个学科调用 canEditSubject()
   ↓
6. 如果不可编辑，设置 disabled=true 和样式
   ↓
7. 用户只能修改可编辑的学科
```

#### 权限检查逻辑
```javascript
canEditSubject(subject) {
    // 1. 检查是否有权限信息
    if (!userPermissions) return false;
    
    // 2. 超级管理员可以编辑所有学科
    if (userPermissions.is_admin) return true;
    
    // 3. 获取学科显示名称（如 'yuwen' -> '语文'）
    const subjectDisplayName = subjectNames[subject];
    
    // 4. 检查是否在可编辑学科列表中
    const editableSubjects = userPermissions.editable_subjects || [];
    return editableSubjects.includes(subjectDisplayName);
}
```

**utils/permission_checker.py**
```python
def can_edit_grade(user, target_class_id, target_subject):
    """检查是否有权限编辑成绩"""
    # 超级管理员可以编辑所有成绩
    if is_super_admin(user):
        return True
    
    # 正班主任可以编辑自己班级的所有成绩
    if is_head_teacher(user) and str(user.class_id) == str(target_class_id):
        return True
    
    # 其他角色只能编辑自己任教学科的成绩
    user_subjects = get_user_subjects(user.id)
    subject_names = [s['name'] for s in user_subjects]
    
    return target_subject in subject_names
```
