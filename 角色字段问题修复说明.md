# 角色字段问题修复说明

## 问题描述

### 问题1：导入时角色被错误覆盖
**现象**：
- 导入Excel时，如果角色列为空
- 系统默认设置为"科任老师"
- 但更新现有用户时，也会把原有角色改成"科任老师"

**影响**：
- 已存在的用户角色被错误覆盖
- 比如原本是"正班主任"的用户，导入后变成了"科任老师"

### 问题2：编辑用户后数据库未更新
**现象**：
- 在管理后台编辑用户角色
- 前端显示已更新
- 但数据库中的数据没有改变

**根本原因**：
- 数据库中同时存在两个角色字段：
  - `role` - 旧字段（默认值'科任老师'）
  - `primary_role` - 新字段（默认值'科任老师'）
- 代码使用的是`primary_role`字段
- 但SQL查看器显示的是`role`字段
- 导致看起来数据没有更新

---

## 解决方案

### 修复1：导入逻辑优化

#### 修改内容
**文件**：`users.py`

**修改点1**：预览和确认导入时，角色字段不再有默认值
```python
# 修改前
role = str(row['角色']).strip() if '角色' in row and pd.notna(row['角色']) else "科任老师"

# 修改后
role = str(row['角色']).strip() if '角色' in row and pd.notna(row['角色']) else ""
```

**修改点2**：只在新增用户时设置默认角色
```python
# 新增用户时，如果没有提供角色，默认为"科任老师"
if not role:
    role = "科任老师"
```

**修改点3**：更新用户时，只更新Excel中提供的字段
```python
# 构建动态更新SQL
update_fields = []
update_params = []

if has_role:  # 只有Excel中提供了角色才更新
    update_fields.append("primary_role = ?")
    update_params.append(role)

if has_class:  # 只有Excel中提供了班级才更新
    update_fields.append("class_id = ?")
    update_params.append(class_id)
```

#### 效果
- ✅ 新增用户：如果角色为空，默认"科任老师"
- ✅ 更新用户：如果角色为空，保持原有角色不变
- ✅ 更新用户：只更新Excel中提供的字段

### 修复2：删除重复的角色字段

#### 问题分析
数据库中有两个角色字段：

#### 修复脚本
**文件**：`fix_role_field.py`

**功能**：

3. 确保所有用户都有primary_role值

## 技术细节

### 为什么会有两个字段？
- 系统升级过程中添加了新的`primary_role`字段
- 但没有删除旧的`role`字段
- 导致数据库中同时存在两个字段

### 为什么前端显示正确但数据库不对？
- 前端读取的是`primary_role`字段（正确）
- 代码更新的也是`primary_role`字段（正确）
- 但SQL查看器默认显示的是`role`字段（旧字段）
- 所以看起来数据没有更新

### 为什么要重建表？
- SQLite不支持直接删除列
- 必须创建新表，复制数据，删除旧表，重命名新表
- 这是SQLite的限制

## 常见问题

### Q1：修复后原有数据会丢失吗？
**答**：不会。修复脚本会先将旧字段的数据复制到新字段，然后才删除旧字段。

### Q2：如果修复失败怎么办？
**答**：
1. 数据库有自动备份
2. 可以从`students.db.backup`恢复
3. 或从`uploads/backups/`目录恢复

### Q3：为什么导入时不能直接覆盖所有字段？
**答**：
- 这样设计是为了灵活性
- 可以只更新部分字段，不影响其他字段
- 比如只更新班级，不改变角色

### Q4：如何批量修改用户角色？
**答**：
1. 准备Excel，只填用户名和新角色
2. 导入时会自动识别为更新操作
3. 只更新角色字段，其他字段保持不变
