# 学科分配 - 拖拽版本 V3.0

## 🎯 设计理念

采用**拖拽式交互**，让学科分配变得像玩游戏一样简单直观！

## ✨ 核心特性

### 1. 拖拽分配
- 将教师卡片拖拽到学科框中
- 实时视觉反馈
- 支持重复拖拽（一个教师可以任教多个学科）

### 2. 可视化管理
- 所有教师卡片显示在顶部
- 所有学科以卡片形式展示
- 每个学科框显示任教的教师

### 3. 快速移除
- 点击教师标签的"×"按钮即可移除
- 确认对话框防止误操作

## 🖱️ 操作方式

### 分配学科（3步）
1. **选择教师** - 在顶部找到要分配的教师
2. **拖拽** - 按住教师卡片，拖到目标学科框
3. **放下** - 松开鼠标，完成分配

✅ 完成！教师立即出现在学科框中。

### 移除学科（2步）
1. **找到教师** - 在学科框中找到要移除的教师
2. **点击×** - 点击教师标签上的"×"按钮

✅ 完成！教师从学科框中移除。

### 批量分配（重复拖拽）
1. 拖拽"王碧霞"到"语文"框
2. 再拖拽"王碧霞"到"数学"框
3. 再拖拽"王碧霞"到"英语"框

✅ 完成！王碧霞现在任教语文、数学、英语三个学科。

## 💡 技术实现

### HTML5 拖拽API
```javascript
// 拖拽开始
card.on('dragstart', function(e) {
    e.originalEvent.dataTransfer.setData('teacherId', teacher.id);
    e.originalEvent.dataTransfer.setData('teacherName', teacher.username);
});

// 拖拽悬停
box.on('dragover', function(e) {
    e.preventDefault();
    $(this).addClass('drag-over');
});

// 放下
box.on('drop', function(e) {
    e.preventDefault();
    const teacherId = e.originalEvent.dataTransfer.getData('teacherId');
    assignTeacherToSubject(teacherId, subjectId);
});
```

### CSS动画
```css
.teacher-card:hover {
    transform: scale(1.05);
    transition: all 0.2s;
}

.subject-box.drag-over {
    transform: scale(1.02);
    transition: all 0.3s;
}
```

### 数据管理
```javascript
// 本地缓存
let teacherSubjects = {}; // {teacherId: [subjectIds]}

// 分配后更新
teacherSubjects[teacherId].push(subjectId);

// 移除后更新
teacherSubjects[teacherId] = teacherSubjects[teacherId].filter(id => id != subjectId);
```

## 📊 性能优化

### 1. 数据预加载
- 一次性加载所有教师的任教信息
- 避免重复API调用
- 使用Promise.all并发加载

### 2. 本地缓存
- 使用JavaScript对象缓存数据
- 操作后立即更新缓存
- 减少服务器请求

### 3. 按需渲染
- 只在切换到"任教分配"标签时加载
- 避免不必要的DOM操作

### 4. 事件委托
- 使用jQuery的事件委托
- 减少事件监听器数量
