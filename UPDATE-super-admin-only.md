## 角色体系

### 当前角色定义
```
1. 超级管理员 (is_admin = 1)
   - 系统最高权限
   - 可访问后台管理
   - 可管理所有用户和班级
   
2. 正班主任 (primary_role = '正班主任')
   - 管理唯一班级
   - 参与绩效考核
   - 无后台管理权限
   
3. 副班主任 (primary_role = '副班主任')
   - 管理唯一班级
   - 不参与绩效考核
   - 无后台管理权限
   
4. 科任老师 (primary_role = '科任老师')
   - 可教多个班级、多个学科
   - 无后台管理权限
   
5. 行政 (primary_role = '行政')
   - 行政人员
   - 无后台管理权限
   
6. 校级领导 (primary_role = '校级领导')
   - 学校领导
   - 无后台管理权限
```

## 使用方法

### 如何设置超级管理员

**方法1：通过界面**
1. 使用现有超级管理员登录
2. 进入后台管理 → 用户管理
3. 编辑用户
4. 勾选"设为超级管理员"
5. 保存

**方法2：通过数据库**
```sql
UPDATE users 
SET is_admin = 1, primary_role = '超级管理员' 
WHERE username = '用户名';
```

### 如何取消超级管理员

**方法1：通过界面**
1. 使用超级管理员登录
2. 进入后台管理 → 用户管理
3. 编辑用户
4. 取消勾选"设为超级管理员"
5. 选择其他角色（如"正班主任"）
6. 保存

**方法2：通过数据库**
```sql
UPDATE users 
SET is_admin = 0, primary_role = '正班主任' 
WHERE username = '用户名';
```

## 安全建议

### 1. 最小权限原则
- 只给必要的人员超级管理员权限
- 定期审查超级管理员列表

### 2. 密码安全
- 超级管理员账号使用强密码
- 定期更换密码
- 不要共享账号

### 3. 操作审计
- 记录超级管理员的操作日志
- 定期检查异常操作

### 4. 备份策略
- 定期备份数据库
- 在进行重大操作前备份

## 常见问题

### Q1: 如何找回超级管理员权限？
**A**: 如果所有超级管理员都无法登录：
```bash
# 通过数据库直接设置
sqlite3 students.db "UPDATE users SET is_admin = 1 WHERE username = 'admin'"
```

### Q2: 可以有多个超级管理员吗？
**A**: 可以。系统支持多个超级管理员，但建议控制数量。

### Q3: 超级管理员可以删除自己吗？
**A**: 不可以。系统禁止删除当前登录的用户。

### Q4: 超级管理员需要指定班级吗？
**A**: 不需要。超级管理员的 `class_id` 可以为空。

### Q5: 超级管理员参与绩效考核吗？
**A**: 不参与。绩效考核只包含 `primary_role = '正班主任'` 且 `is_admin = 0` 的用户。
