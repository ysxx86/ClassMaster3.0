# 修复班级学科权限精确匹配

## 问题描述

郑一夫老师的分配：
- 1年级1班、2班：书法
- 1年级3班、4班：道法

**问题**：郑老师可以编辑1班、2班的道法，也可以编辑3班、4班的书法

**原因**：权限检查逻辑只检查了"是否任教该学科"和"是否任教该班级"，没有精确匹配"是否任教该班级的该学科"

## 解决方案

### 1. 后端修改

#### 修改 `utils/permission_checker.py`

**`can_edit_grade()` 函数**：
- 原来：检查教师是否任教该学科（从 teacher_subjects 表）
- 现在：精确检查教师是否任教该班级的该学科（从 teaching_assignments 表）

```python
# 其他角色：必须精确匹配班级和学科
cursor.execute('''
    SELECT COUNT(*) as count
    FROM teaching_assignments
    WHERE teacher_id = ? AND class_id = ? AND subject = ?
''', (user.id, str(target_class_id), target_subject))
```

**`get_editable_subjects()` 函数**：
- 原来：返回教师任教的所有学科
- 现在：返回教师在指定班级任教的学科

```python
# 其他角色：从 teaching_assignments 表获取该班级的任教学科
cursor.execute('''
    SELECT DISTINCT subject
    FROM teaching_assignments
    WHERE teacher_id = ? AND class_id = ?
''', (user.id, str(class_id)))
```

**`get_user_permissions()` 函数**：
- 新增：返回 `teaching_map`（班级-学科映射）

```python
teaching_map = {
    '56': ['书法'],  # 1年级1班
    '57': ['书法'],  # 1年级2班
    '58': ['道法'],  # 1年级3班
    '59': ['道法']   # 1年级4班
}
```

### 2. 前端修改

#### 修改 `js/grades.js`

**`canEditSubject()` 函数**：
- 原来：只检查学科
- 现在：检查班级和学科的组合

```javascript
// 检查 teaching_map 中是否有该班级的该学科
const teachingMap = userPermissions.teaching_map || {};
const classSubjects = teachingMap[String(classId)] || [];
return classSubjects.includes(subjectDisplayName);
```

**`renderGradesTable()` 函数**：
- 调用 `canEditSubject()` 时传入班级ID

```javascript
const canEdit = canEditSubject(subject, studentGrade.class_id);
```

## 权限检查流程

### 正班主任
1. 检查角色是否为"正班主任"
2. 检查班级ID是否匹配
3. 如果匹配，可以编辑该班级的所有学科

### 其他角色（科任老师、副班主任等）
1. 从 `teaching_map` 获取该班级的任教学科列表
2. 检查目标学科是否在列表中
3. 只有在列表中才允许编辑
