# 角色系统更新完成

## 更新内容

### 1. 角色系统扩展 ✅
从原来的2个角色（管理员/班主任）扩展到6个角色：
- 正班主任
- 副班主任  
- 科任老师
- 行政
- 校级领导
- 超级管理员

### 2. 数据库更新 ✅
- `users` 表添加 `role` 字段
- 默认值：'科任老师'
- 已迁移旧的 `teacher_type` 数据

### 3. 后台管理界面更新 ✅
**文件**: `pages/admin.html`
- 用户列表显示角色徽章（不同颜色）
- 编辑用户模态框添加角色下拉选择
- 显示班级名称而非班级ID

**文件**: `js/admin.js?v=20260117`
- 渲染用户列表时显示角色
- 更新用户时发送role字段
- 添加版本号防止缓存

**文件**: `users.py`
- GET `/api/users` 返回role字段
- PUT `/api/users/<id>` 接受并保存role字段
- 添加调试日志

### 4. 绩效考核系统更新 ✅
**文件**: `performance.py`
- 只显示"正班主任"参与考核
- 排除管理员账号
- 排除其他角色（副班主任、科任老师、行政、校级领导）

**关键查询**:
```python
WHERE u.role = '正班主任' AND u.is_admin = 0
```

## 浏览器缓存问题

如果页面没有更新，请：

### 方法1: 强制刷新（推荐）
- **Windows/Linux**: `Ctrl + Shift + R`
- **Mac**: `Cmd + Shift + R`

### 方法2: 清除缓存
1. 打开浏览器设置
2. 清除浏览数据
3. 选择"缓存的图片和文件"
4. 清除并刷新页面

### 方法3: 无痕模式
- **Chrome/Edge**: `Ctrl + Shift + N` (Windows) 或 `Cmd + Shift + N` (Mac)
- **Firefox**: `Ctrl + Shift + P` (Windows) 或 `Cmd + Shift + P` (Mac)

## 技术细节

### API端点

#### 获取用户列表
```
GET /api/users
返回: 包含role字段的用户列表
```

#### 更新用户
```
PUT /api/users/<user_id>
请求体: {
  "username": "张老师",
  "class_id": "5",
  "role": "副班主任",
  "is_admin": false
}
```

#### 获取绩效考核矩阵
```
GET /api/performance/scores/<semester>
返回: 只包含正班主任的评分矩阵
```

### 数据库字段

```sql
-- users表
role TEXT DEFAULT '科任老师'

-- 有效值
'正班主任', '副班主任', '科任老师', '行政', '校级领导', '超级管理员'
```

### 前端角色徽章颜色
