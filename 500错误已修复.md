# ✅ 500错误已修复!

## 🔍 问题原因

### 错误信息
```
realtime_api - ERROR - 检查更新失败: no such table: grades
GET /api/check-updates?since=... HTTP/1.1" 500
```

### 根本原因
- `realtime_api.py` 尝试查询 `grades` 表
- 但数据库中**没有 `grades` 表**
- 导致SQL查询失败,返回500错误

## ✅ 解决方案

### 已修改的文件
- `realtime_api.py` - 添加了表存在性检查

### 修改内容
添加了 `table_exists()` 函数,在查询前检查表是否存在:

```python
# 检查表是否存在的辅助函数
def table_exists(table_name):
    cursor.execute("""
        SELECT name FROM sqlite_master 
        WHERE type='table' AND name=?
    """, (table_name,))
    return cursor.fetchone() is not None

# 只有表存在时才检查
if table_exists('students'):
    # 检查学生表更新
    
if table_exists('grades'):
    # 检查成绩表更新
```

### 改进点
1. ✅ 查询前检查表是否存在
2. ✅ 即使某个表不存在,也不会报错
3. ✅ 错误时返回空更新列表,而不是500错误
4. ✅ 使用 `logger.debug` 记录非关键错误

## 💡 技术说明

### 修改前的问题
```python
# 直接查询,如果表不存在会报错
cursor.execute('SELECT COUNT(*) FROM grades WHERE ...')
```

### 修改后的解决方案
```python
# 先检查表是否存在
if table_exists('grades'):
    try:
        cursor.execute('SELECT COUNT(*) FROM grades WHERE ...')
    except Exception as e:
        logger.debug(f"检查失败: {e}")  # 静默失败
```

### 优点
1. **健壮性**: 不会因为表不存在而崩溃
2. **兼容性**: 适配不同的数据库结构
3. **可维护性**: 容易添加新的表检查
4. **用户体验**: 不影响正常使用
