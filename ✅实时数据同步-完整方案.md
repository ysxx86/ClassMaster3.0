# 实时数据同步 - 完整方案

## 问题描述

**用户需求：**
- 在任教信息弹窗修改数据后，数据库已更新
- 成绩管理页面应该**自动重新查询数据库**
- 获取最新数据并更新界面
- **不需要手动刷新页面**

**核心问题：**
这不是权限问题，而是**数据同步**问题。当一个页面修改了数据，其他页面需要知道数据已变化，并自动重新加载。

## 实现方案

### 方案架构

```
任教信息弹窗（修改数据）
    ↓
保存到数据库 ✅
    ↓
触发更新通知
    ├─ 方式1: CustomEvent（同一窗口）
    ├─ 方式2: localStorage（跨标签页/iframe）
    └─ 方式3: 轮询检查（兜底方案）
    ↓
成绩管理页面监听到通知
    ↓
重新查询数据库
    ├─ 1. 获取最新权限
    └─ 2. 加载最新成绩数据
    ↓
更新界面 ✅
```

### 实现细节

#### 1. 任教信息保存时触发通知

**文件：** `js/teacher-info-confirmation.js`

```javascript
async function saveChanges() {
    try {
        // 1. 保存到数据库
        const updateResponse = await fetch('/api/teacher-info/update', {
            method: 'POST',
            body: JSON.stringify({ subjects, assignments })
        });
        
        // 2. 确认状态
        await fetch('/api/teacher-info/confirm', {
            method: 'POST',
            body: JSON.stringify({ info: {...} })
        });
        
        // 3. ⭐ 触发更新通知（三种方式）
        
        // 方式1: CustomEvent（同一窗口内的通信）
        window.dispatchEvent(new CustomEvent('teacherInfoUpdated', {
            detail: {
                subjects: subjectIds,
                assignments: assignments,
                timestamp: Date.now()
            }
        }));
        
        // 方式2: localStorage（跨标签页/iframe通信）
        localStorage.setItem('teacherInfoUpdateTimestamp', Date.now().toString());
        
        // 4. 显示成功提示
        showSuccessToast('任教信息保存成功！正在更新页面...');
        
    } catch (error) {
        showErrorToast('保存失败: ' + error.message);
    }
}
```

#### 2. 成绩管理页面监听更新

**文件：** `js/grades.js`

```javascript
document.addEventListener('DOMContentLoaded', function() {
    // 初始化...
    
    // ⭐ 方式1: 监听CustomEvent（同一窗口）
    window.addEventListener('teacherInfoUpdated', function(e) {
        console.log('✅ 检测到教师信息更新，立即重新加载数据...', e.detail);
        
        // 显示提示
        showNotification('任教信息已更新，正在刷新数据...', 'info');
        
        // 重新获取权限并加载数据
        fetchUserPermissions().then(() => {
            loadGrades(); // ⭐ 重新从数据库加载
            showNotification('数据已更新！', 'success');
        });
    });
    
    // ⭐ 方式2: 监听localStorage变化（跨标签页/iframe）
    window.addEventListener('storage', function(e) {
        if (e.key === 'teacherInfoUpdateTimestamp') {
            console.log('✅ 检测到其他页面更新了教师信息');
            
            fetchUserPermissions().then(() => {
                loadGrades(); // ⭐ 重新从数据库加载
                showNotification('数据已同步更新！', 'success');
            });
        }
    });
    
    // ⭐ 方式3: 定期轮询检查（兜底方案）
    let lastTeacherInfoUpdate = localStorage.getItem('teacherInfoUpdateTimestamp') || '0';
    
    setInterval(() => {
        const currentUpdate = localStorage.getItem('teacherInfoUpdateTimestamp') || '0';
        
        if (currentUpdate !== lastTeacherInfoUpdate && currentUpdate !== '0') {
            console.log('✅ 检测到教师信息更新（轮询检查）');
            lastTeacherInfoUpdate = currentUpdate;
            
            fetchUserPermissions().then(() => {
                loadGrades(); // ⭐ 重新从数据库加载
                showNotification('数据已自动更新！', 'success');
            });
        }
    }, 2000); // 每2秒检查一次
});
```

#### 3. loadGrades() 函数重新查询数据库

**文件：** `js/grades.js`

```javascript
function loadGrades() {
    const gradesTable = document.querySelector('.grades-table table tbody');
    
    // 显示加载状态
    gradesTable.innerHTML = `<tr><td colspan="15">加载中...</td></tr>`;
    
    // ⭐ 从API重新获取数据（查询数据库）
    fetch(`/api/grades?semester=${currentSemester}`)
        .then(response => response.json())
        .then(data => {
            if (data.status === 'ok') {
                // ⭐ 渲染最新数据
                renderGradesTable(data.grades || []);
            }
        });
}
```

## 三种通知方式对比

### 方式1: CustomEvent

**适用场景：** 同一窗口内的不同组件通信

**优点：**
- 实时性最好（立即触发）
- 可以传递详细数据
- 不依赖外部存储

**缺点：**
- 只能在同一窗口内工作
- 不能跨标签页
- 不能跨iframe（除非手动传递）

**代码：**
```javascript
// 发送
window.dispatchEvent(new CustomEvent('teacherInfoUpdated', {
    detail: { subjects, assignments }
}));

// 接收
window.addEventListener('teacherInfoUpdated', function(e) {
    console.log('收到更新:', e.detail);
});
```

### 方式2: localStorage + storage事件

**适用场景：** 跨标签页通信

**优点：**
- 可以跨标签页
- 浏览器原生支持
- 自动触发storage事件

**缺点：**
- 同一标签页内不触发storage事件
- 只能存储字符串
- 有存储大小限制

**代码：**
```javascript
// 发送
localStorage.setItem('teacherInfoUpdateTimestamp', Date.now().toString());

// 接收（只在其他标签页触发）
window.addEventListener('storage', function(e) {
    if (e.key === 'teacherInfoUpdateTimestamp') {
        console.log('其他标签页更新了数据');
    }
});
```

### 方式3: 轮询检查localStorage

**适用场景：** 兜底方案，确保所有情况都能检测到

**优点：**
- 可靠性最高
- 适用于所有场景（同标签页、跨标签页、iframe）
- 不依赖事件触发

**缺点：**
- 有延迟（2秒）
- 消耗一定性能
- 需要维护状态

**代码：**
```javascript
let lastUpdate = '0';

setInterval(() => {
    const currentUpdate = localStorage.getItem('teacherInfoUpdateTimestamp') || '0';
    
    if (currentUpdate !== lastUpdate && currentUpdate !== '0') {
        lastUpdate = currentUpdate;
        console.log('检测到更新（轮询）');
        // 重新加载数据
    }
}, 2000);
```

## 数据流程

### 完整流程图

```
用户操作：修改任教信息
    ↓
前端：调用 /api/teacher-info/update
    ↓
后端：更新数据库
    ├─ teacher_subjects 表
    └─ teaching_assignments 表
    ↓
前端：触发三种通知
    ├─ CustomEvent ────────────→ 同一窗口的成绩管理页面
    ├─ localStorage ───────────→ 其他标签页的成绩管理页面
    └─ localStorage（轮询）────→ iframe中的成绩管理页面
    ↓
成绩管理页面：检测到更新
    ↓
1. 调用 fetchUserPermissions()
    ├─ 请求 /api/user/permissions
    ├─ 获取最新权限数据
    └─ 更新本地权限缓存
    ↓
2. 调用 loadGrades()
    ├─ 请求 /api/grades?semester=xxx
    ├─ 从数据库查询最新成绩
    └─ 获取最新数据
    ↓
3. 调用 renderGradesTable(data)
    ├─ 清空旧表格
    ├─ 渲染新数据
    └─ 更新界面
    ↓
用户看到：最新的成绩数据 ✅
```

### 时间线

```
T0: 用户点击"保存修改"
T0+100ms: 数据库更新完成
T0+150ms: 触发CustomEvent（同一窗口立即收到）
T0+150ms: 写入localStorage
T0+200ms: storage事件触发（其他标签页收到）
T0+300ms: 开始重新加载数据
T0+500ms: 数据加载完成，界面更新 ✅
T0+2000ms: 轮询检查（兜底，确保iframe也收到）
```

## 性能优化

### 1. 防抖处理

如果短时间内多次修改，避免重复加载：

```javascript
let updateTimer = null;

window.addEventListener('teacherInfoUpdated', function(e) {
    // 清除之前的定时器
    if (updateTimer) {
        clearTimeout(updateTimer);
    }
    
    // 延迟500ms执行，避免频繁刷新
    updateTimer = setTimeout(() => {
        fetchUserPermissions().then(() => {
            loadGrades();
        });
    }, 500);
});
```

### 2. 缓存权限数据

避免每次都重新获取权限：

```javascript
let permissionsCache = null;
let permissionsCacheTime = 0;

async function fetchUserPermissions() {
    const now = Date.now();
    
    // 如果缓存未过期（5秒内），直接使用缓存
    if (permissionsCache && (now - permissionsCacheTime) < 5000) {
        return permissionsCache;
    }
    
    // 重新获取
    const response = await fetch('/api/user/permissions');
    const data = await response.json();
    
    permissionsCache = data;
    permissionsCacheTime = now;
    
    return data;
}
```

### 3. 只更新变化的部分

不重新渲染整个表格，只更新变化的列：

```javascript
function updateGradesColumns(newSubjects) {
    // 对比新旧学科列表
    const oldSubjects = getCurrentSubjects();
    const added = newSubjects.filter(s => !oldSubjects.includes(s));
    const removed = oldSubjects.filter(s => !newSubjects.includes(s));
    
    // 只添加新列
    added.forEach(subject => {
        addSubjectColumn(subject);
    });
    
    // 只删除旧列
    removed.forEach(subject => {
        removeSubjectColumn(subject);
    });
}
```

## 注意事项

### 1. localStorage的限制

- 大小限制：通常5-10MB
- 只能存储字符串
- 同步操作，可能阻塞主线程

**解决方案：** 只存储时间戳，不存储大量数据

### 2. 轮询的性能影响

- 每2秒执行一次检查
- 可能影响性能

**解决方案：**
- 只在成绩管理页面启用轮询
- 页面不可见时停止轮询

```javascript
document.addEventListener('visibilitychange', function() {
    if (document.hidden) {
        // 页面隐藏，停止轮询
        clearInterval(pollingTimer);
    } else {
        // 页面显示，恢复轮询
        startPolling();
    }
});
```

### 3. 多次触发的问题

三种方式可能导致同一个更新被处理多次

**解决方案：** 使用时间戳去重

```javascript
let lastProcessedUpdate = 0;

function handleUpdate(timestamp) {
    if (timestamp <= lastProcessedUpdate) {
        console.log('重复的更新，忽略');
        return;
    }
    
    lastProcessedUpdate = timestamp;
    
    // 处理更新
    fetchUserPermissions().then(() => {
        loadGrades();
    });
}
```
