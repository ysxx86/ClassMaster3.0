# 班主任绩效考核管理系统

## 系统概述

班主任绩效考核管理系统是德育AI智能系统的一个重要模块，用于科学、公平、公正地评价班主任的工作绩效。

## 功能特点

### 1. 教师类型管理
- 支持区分**正班主任**和**副班主任**
- 管理员可以在"教师管理"标签中设置教师类型
- 考核主要针对正班主任进行

### 2. 考核项目管理（管理员功能）
系统预设了以下考核类别和项目：

#### 计划总结（10%）
- 计划总结10%：班级工作计划和总结

#### 常规教育（30%）
- 校行评语5%：校行评语
- 班队课5%：班队课开展情况
- 安全工作5%：安全教育工作
- 礼仪卫生5%：礼仪卫生管理
- 环境卫生10%：环境卫生管理

#### 班级建设（15%）
- 班级建设5%：班级文化建设
- 班级布置5%：班级布置
- 共性成长5%：学生共性成长

#### 活动开展（15%）
- 活动开展10%：班级活动开展
- 大课间活动5%：大课间活动组织

#### 家校配合（15%）
- 家校配合5%：家校沟通配合
- 家校活动5%：家校活动开展
- 家校群管理5%：家校群管理

#### 工作态度（10%）
- 工作态度10%：工作态度评价

#### 德育经验（3%）
- 德育经验3%：德育经验交流

#### 承担任务（3%）
- 承担任务3%：承担额外任务

管理员可以：
- 添加新的考核项目
- 编辑现有项目的权重和说明
- 删除不需要的项目（软删除）

### 3. 评分人员管理（管理员功能）
- 管理员可以指定哪些用户有权限进行评分
- 可以设置评分人员对特定项目的评分权限
- 可以设置评分人员对所有项目的评分权限

### 4. 评分录入
- 评分人员可以对被评教师的各个考核项目进行评分
- 支持按学期进行评分
- 每个项目可以由多人评分
- 评分范围：0-100分
- 支持添加评分备注

### 5. 自动计算规则
- 当某个考核项目有**3个或以上评分**时，系统会自动**去掉一个最高分和一个最低分**
- 剩余评分取平均值作为该项目的最终得分
- 最终总分 = Σ(各项目平均分 × 权重)

### 6. 考核结果查询
- 显示所有教师的考核总分和排名
- 支持按学期查询
- 前三名会有特殊的奖牌标识（🥇🥈🥉）
- 显示教师类型、班级等信息

## 使用流程

### 管理员操作流程

1. **设置教师类型**
   - 进入"教师管理"标签
   - 为每位教师设置"正班主任"或"副班主任"
   - 点击"保存"按钮

2. **配置评分人员**
   - 进入"评分人员"标签
   - 点击"添加评分人员"
   - 选择用户并设置评分权限
   - 可以指定特定项目或全部项目

3. **管理考核项目**（可选）
   - 进入"考核项目"标签
   - 可以添加、编辑或删除考核项目
   - 调整项目权重

4. **计算考核结果**
   - 进入"考核结果"标签
   - 选择学期
   - 点击"计算结果"按钮
   - 系统自动计算并显示排名

### 评分人员操作流程

1. **选择评分对象**
   - 进入"评分录入"标签
   - 选择学期
   - 选择要评分的教师

2. **录入评分**
   - 在各个考核项目中输入分数（0-100）
   - 可以添加评分备注
   - 点击"保存"按钮

3. **查看结果**
   - 进入"考核结果"标签
   - 选择学期
   - 查看所有教师的考核结果和排名

## 数据库结构

### 1. users表扩展
```sql
ALTER TABLE users ADD COLUMN teacher_type TEXT DEFAULT '正班主任'
```

### 2. performance_items（考核项目表）
- id: 主键
- category: 类别
- item_name: 项目名称
- weight: 权重（百分比）
- description: 说明
- is_active: 是否启用

### 3. performance_evaluators（评分人员表）
- id: 主键
- user_id: 用户ID
- item_id: 项目ID（NULL表示所有项目）
- can_evaluate_all: 是否可以评所有项目

### 4. performance_scores（评分记录表）
- id: 主键
- teacher_id: 被评教师ID
- item_id: 考核项目ID
- evaluator_id: 评分人ID
- score: 分数
- semester: 学期
- comments: 备注
- 唯一约束：(teacher_id, item_id, evaluator_id, semester)

### 5. performance_results（考核结果表）
- id: 主键
- teacher_id: 教师ID
- semester: 学期
- total_score: 总分
- rank: 排名
- 唯一约束：(teacher_id, semester)

## API接口

### 考核项目管理
- GET `/api/performance/items` - 获取所有考核项目
- POST `/api/performance/items` - 添加考核项目
- PUT `/api/performance/items/<id>` - 更新考核项目
- DELETE `/api/performance/items/<id>` - 删除考核项目

### 评分人员管理
- GET `/api/performance/evaluators` - 获取所有评分人员
- POST `/api/performance/evaluators` - 添加评分人员
- DELETE `/api/performance/evaluators/<id>` - 删除评分人员

### 评分管理
- POST `/api/performance/scores` - 提交评分
- GET `/api/performance/scores/<teacher_id>/<semester>` - 获取教师评分

### 结果管理
- POST `/api/performance/calculate/<semester>` - 计算考核结果
- GET `/api/performance/results/<semester>` - 获取考核结果

### 教师管理
- GET `/api/performance/teachers` - 获取所有教师
- PUT `/api/performance/teachers/<id>/type` - 更新教师类型

### 核心算法
```python
# 去掉最高最低分后取平均
if len(scores) >= 3:
    scores = scores[1:-1]  # 去掉首尾（最低和最高）
avg_score = sum(scores) / len(scores)
total_score += avg_score * weight / 100
```

## 注意事项

1. **权限管理**
   - 只有管理员可以管理考核项目、评分人员和计算结果
   - 评分人员只能评分自己有权限的项目
   - 所有用户都可以查看考核结果

2. **数据完整性**
   - 评分必须在0-100之间
   - 每个评分人对同一教师的同一项目在同一学期只能评一次分
   - 删除考核项目是软删除，不会影响历史数据

3. **计算规则**
   - 只有3个或以上评分才会去掉最高最低分
   - 如果只有1-2个评分，直接取平均值
   - 总分保留两位小数

4. **学期管理**
   - 学期格式：YYYY-YYYY-N（如：2024-2025-1）
   - 支持多学期数据并存
   - 每个学期独立计算排名
